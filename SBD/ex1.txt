1. Defina e correlacione os conceitos de banco de dados, sistema de banco de dados e de sistema gerenciador de banco de dados.
	R: O conceito de banco de dados, é basicamente um aglomerado de informações reunidas e diretamente relacionadas, que sao armazenadas sobre um dominio específico, ja 
o SBD é um sitema de regitros envolvendo quatro componentes principais, hardware, software e usuario, um SBD possui como objetivo isolar o usuario de datalhes 
mais internos do BD(abstração de dados) e prover independencia de dados as aplicações, com o SGBD é um pouco diferente, pois é um software com recursos específicos para 
facilitar a manipulação de informações de um BD e o desenvolivmento de programas aplicativos como Oracle, Parados, MySQL, Interbase, Sybase, etc..

2. Como os SGBD garantem independência e consistência de dados?
	R: Através dos mecanismos de roll backs por exemplo, para evitar que um estado de inconsistencia seja realizado o roll back devolve a base de dados ao seu estado de 
consistência. E a independência, e garantido porque cada aplicação vê apenas os dados que lhe interessam, Ela nao precisa saber como os dados estão fisicamente armazenados, 
e tambem nao precisa ser modificada caso a estrutura de dados seja alterada.

3. Discuta as vantagens de um SBD quando comparado com um sistema de arquivos.
    R: Quando possuir um conjunto muito grande de dados, é inviavel usar um sistema local de arquivos, pois um SBD possui diversas vantagens como independencia de arquivos, 
ou seja um SBD nao precisa conhecer as estruturas dos dados, nao precisa saber detalhes de como seus dados estao fisicamente armazenados e também possui Redundancia 
controlada,os dados armazenados possuem consistencia e segurança entre muitas outras vantagens que superam os arquivos locais convencionais, porem um sistema de arquivos para 
dados pequenos costuma ser mais vantajoso pela questão de custo e facilidade de gerenciamente.

4. Descreva como ocorre a abstração de dados em cada um dos níveis da arquitetura em três níveis (three-schema) dos sistemas de banco de dados.
    R: Uma vez que muitos dos usuários de banco de dados não são treinados para computação, a complexidade está escondida deles através de diversos níveis de abstração que
simplificam a interação do usuário com o sistema. No nivel mais baixo (Nivel Físico) esta a complexidade de como os dados estão armazenados realmente

5. Por que a independência lógica é mais difícil de ser obtida do que a independência física?
    R: Porque ela modifica o esquema conceitual de um Banco de Dados a fim de alterar por exemplo a estrutura de um BD, o que pode acarretar em erros graves na hora de 
criar e armazenar essas estruturas novas.

6. Diferencie instância e esquema.
    R:Um esquema é especificado durante o projeto da base de dados com diagramas de entidades e relacionamentos, nao sofre muitas alterações, ja as instâncias são
são dados presentes em qualquer ponto do tempo de um Banco de Dados que estão sujeitos a mudanças frequentes.

7. Qual a diferença entre um esquema de banco de dados e um estado de banco de dados?
    R: Um esquema é o projeto em si de como um BD se comportará, e um Estado são as fases atuais e os comportamentos do Banco de Dados depois que foi criado, ou seja, é o
funcionamento na pratica depois de projetado.

8. Qual a diferença entre DDL e DML?
    R: DDL e DML sao tipos de linguagens SQL. A linguagem SQL e divididade entre esses dois tipos. A principal diferença entre eles é que DDL esta mais relacionado com a
estrutura de uma tabela, enquanto que DML manipula os registros que ficam nesta tabela.

9. Analisando a estrutura de componentes do SGBD dada em aula, explique o que ocorre em cada componente e como é o fluxo de dados/informações entre os componentes.
    R: 
    * Componentes para o tratamento de consultas: Executa instruções de baixo nível geradas pelo compilador DML.
    * Gerenciamento de transações: Garante a consistência do banco de dados a despeito de falhas no sistema, além de garantir que transações concorrentes serão executadas 
sem conflitos em seus procedimentos.
    * Compilador DDL: Interpreta os comandos DDL e os registra em um conjunto de tabelas que contêm o Dicionário de Dados.
    * Gerenciador de Buffer: Gerencia a troca de informações entre o disco e a memória principal. Decide ainda quais dados colocar em cache.
    * Gerenciador de Arquivos: Gerencia a alocação de espaços no armazenamento em disco e as estruturas de dados usadas para representar as informações armazenadas.
    * Gerenciador de Armazenamento: Arquivo que retém os dados propriamente dito.
    * Armazenamento: Arquivo que retém os dados propriamente dito.
    * Recuperação e Log: Contém informações estatísticas relativas aos dados contidos no banco de dados. Usadas pelo processador de consultas para seleção de meios eficientes 
para execução das consultas.
    * Controle de Recorrencia: Proporcionam acesso rápido aos itens de dados que são associados a valores
determinados.
